Train a Recommender system ^dOQRm5AA

This model will be able to predict a user's interest in a given question or topic if they have not already 
rated that question. A decay mechanism will be put in place to take into account how old a rating is, since 
a person's interests do change, often drastically, over long periods of time. ^l5YGOh8l

Train a Multi-Armed Bandit ^xQm6atvY

The multi-armed bandit will be a reinforcement learning model that takes into account the previous models predictions,
How likely is the user to get the question correct, how well do they understand the topic, and how much interest to do
they have in the question itself. A reward function will be spelled out, to maximize both accuracy and confidence, with a
penalty function for questions that a user is not interested in, and a penalty for selecting questions the user does not 
understand at all. The MAB here will serve as the replacement for the current PresentationSelectionWorker, this model 
will decide which questions get shown to the user and when. ^wFOYAnM4

The MAB will only be allowed to select from questions in the graph ^rOIx5wyv

A CirculationWorker mechanic decides what questions out of all possibilities will be currently eligble for the MAB
to select from, it trained to add new questions to the active cirriculum that are near (or adjacent/nearby to) questions 
that are already circulating, ensuring that available questions are at least similar to each other on the map. This also
serves to resolve the issue of how to map a user's current existing knowledge.
This mechanism is also trained to select new cirriculum items based on what has the highest probability of being answered.
We could train a dedicated model that works on deciding just what to add to the active cirriculum or not, and what to remove
if items do need to be removed.  ^qoHdFXTu
